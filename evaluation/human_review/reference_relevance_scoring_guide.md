# 人工引用相关性打分说明

评分对象：每条测试用例召回的 Top 3 知识库引用。

评分目的：判断 RAG 返回的案例是否真的能支撑用户 Brief，而不是只看是否有返回结果。

## 打分口径

| 分数 | 判断标准 | 处理建议 |
|---:|---|---|
| 5 | 引用同时匹配内容类型、用户场景、产品特性和表达结构 | 可作为优质案例保留 |
| 4 | 主要方向匹配，但某个特性或语气不完全贴合 | 可用，后续补更精准案例 |
| 3 | 只提供通用结构或泛化表达，和 Brief 部分相关 | 标记观察，必要时补库 |
| 2 | 相关性较弱，可能把生成内容带偏 | 进入知识库补充待办 |
| 1 | 不相关、冲突或存在事实/合规风险 | 优先排查 payload、标签和检索策略 |

## 验收规则

- 每条 case 至少检查 Top 3 引用。
- `human_relevance_score_1_5` 填 1-5 的整数。
- `human_relevance_label` 建议填写：`高度相关`、`基本相关`、`部分相关`、`弱相关`、`不相关`。
- 若 Top 3 平均分低于 4，或任一引用低于 3，需要在 `action` 中标记 `补库`、`改标签`、`调检索` 或 `移除风险内容`。
- 低分原因会进入知识库治理复盘，用于补充 Qdrant 案例、产品事实、品牌规范或合规规则。
