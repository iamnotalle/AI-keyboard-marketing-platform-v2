# 产品迭代路线图

## V0：基础生成器

目标：证明输入 Brief 后可以生成 Blog / EDM。

- 表单输入产品信息。
- 调用 LLM 生成英文内容。
- 支持下载内容。

主要问题：

- 缺少引用依据。
- 容易编造事实。
- 没有审核闭环。

## V1：RAG 可解释生成

目标：让生成内容有知识库依据。

- 接入 Qdrant Cloud。
- 检索历史 Blog / EDM 案例。
- 展示 Top 3 引用案例。
- 加入产品事实、品牌语气、合规规则。

衡量指标：

- Top 3 引用返回率。
- 引用相关率。
- 事实一致性通过率。

## V2：双检双审闭环

目标：从“生成工具”升级为“可控内容生产流程”。

- 草稿 Agent。
- 一审 Agent。
- 迭代 Agent。
- 终审 Agent。
- 人审意见。
- 固定审核维度。

衡量指标：

- 终审平均分。
- 一审到终审提升。
- 合规风险通过率。
- 品牌安全通过率。

## V3：批量测评与质量看板

目标：让产品质量可以被持续衡量。

- 建立 30 条测试集。
- 覆盖 normal / edge / risk。
- 建立 rules / api 两种测评模式。
- 输出 CSV 和 JSONL 报告。
- 增加按类别、内容类型、风险类型的分析。
- 增加人工引用相关性打分表。
- 增加 API 模式小样本报告。

衡量指标：

- 测试集通过率。
- Risk 用例通过率。
- API fallback 率。
- 平均生成耗时。
- Top 3 人工引用相关性平均分。

## V4：运营闭环

目标：让线上使用数据反哺产品。

- 记录生成次数、下载率、Tab 查看率。
- 统计低分维度 Top 3。
- 人工审核通过内容回流知识库。
- 为知识库新增质量标签。
- 对低分维度建立知识库补充待办。
- 将线上真实失败案例回流到测试集。

衡量指标：

- 人工审核通过率。
- 最终下载率。
- 知识库回流数量。
- 高频失败原因。
- 失败案例修复后的回归通过率。

## V5：业务化扩展

目标：从面试 Demo 扩展到真实团队工具。

- 支持品牌词库后台维护。
- 支持多产品线。
- 支持多语言市场。
- 支持团队协作和审核状态。
- 支持接入真实邮件工具或 CMS。

当前项目处于 V2 + V3 阶段，并补齐了 V4 的基础运营闭环：已经实现可体验 Demo、RAG、双检双审、批量测评、人工引用打分、低分补库待办和失败案例回流模板。
