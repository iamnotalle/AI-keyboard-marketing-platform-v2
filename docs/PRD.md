# PRD：键盘出海 AI 营销平台

## 1. 产品背景

海外键盘品牌在内容营销中需要持续产出 Blog、EDM 等英文内容。传统流程依赖人工查历史案例、写初稿、做品牌和合规审核，效率低且质量不稳定。单纯使用通用大模型虽然可以快速生成文本，但容易出现事实编造、竞品词、夸大承诺、EDM 合规缺口等问题。

本项目把“AI 文案生成”设计成一个可控的营销内容生产流程：用户输入产品信息和营销需求后，系统从知识库检索引用案例，生成草稿，完成一审、迭代和终审，并输出最终稿、Top 3 引用案例、审核维度和人审意见。

## 2. 目标用户

- 海外营销运营：需要快速产出 Blog / EDM 初稿。
- 内容运营：需要复用历史案例和品牌表达。
- 产品经理 / 品牌负责人：需要确认文案是否符合产品事实和品牌边界。
- 合规或审核人员：需要快速定位风险表达和人工复核重点。

## 3. 用户问题

- 写作效率低：每次写 Blog / EDM 都要重新查资料、搭结构。
- 历史案例难复用：案例散落在文档和历史活动中，无法直接转成生成依据。
- 生成内容不可控：通用 LLM 可能编造参数、认证、保修、用户评价。
- 审核标准不稳定：不同人对品牌安全、合规风险和 CTA 的判断不一致。
- 面试或 Demo 场景中，用户不应看到 API key、底层配置或复杂技术控件。

## 4. 产品目标

| 目标 | 说明 |
|---|---|
| 可用 | 输入 Brief 后可以生成可读的英文 Blog / EDM |
| 可解释 | 展示知识库 Top 3 引用案例，并支持用户反馈引用是否相关 |
| 可审核 | 固定审核维度，面向用户输出发布建议，面向 PM 后台输出低分诊断 |
| 可迭代 | 草稿一审后自动修改，再展示改前改后对比 |
| 可控 | 拦截竞品词、虚构事实、绝对化和医疗化表达 |
| 可评估 | 通过测试集和测评脚本量化质量 |

## 5. 核心范围

当前版本聚焦两个内容类型：

- Blog
- EDM

不纳入当前版本的内容类型：

- 社媒帖
- 广告短句
- 产品详情页
- 视频脚本

原因：当前知识库中 Blog / EDM 的案例、结构和审核规则最完整，先聚焦高质量闭环。

## 6. 用户故事

| 角色 | 用户故事 | 验收标准 |
|---|---|---|
| 营销运营 | 我想输入产品名、特性和营销需求，快速获得英文内容初稿 | 生成内容包含产品名、核心特性和 CTA |
| 内容运营 | 我想看到引用了哪些案例，并反馈是否相关 | 引用案例展示 Top 3，且支持采用、不采用、不相关标记 |
| 品牌负责人 | 我想知道内容是否有品牌安全风险 | 审核结果包含品牌安全维度和建议 |
| 合规人员 | 我想快速判断是否有夸大或虚构事实 | 审核结果包含合规风险、事实一致性和人审意见 |
| PM / 内容负责人 | 我想看清楚 Agent 修改了什么 | 页面展示初稿问题、一审建议、终稿和提升维度 |
| PM / 运营负责人 | 我想知道低分要怎么处理 | PM 后台展示低分原因、资料缺口和内部迭代动作 |
| PM 面试官 | 我想直接体验产品，不想配置 key | 线上页面不显示 API 输入框 |

## 7. 核心功能

### 7.1 Brief 输入

- 产品名称
- 产品特性
- 内容类型：Blog / EDM
- 营销目标
- 文案语气
- 内容长度
- 内容结构
- 避免出现的词
- 营销需求
- 附加提示词

### 7.2 知识库检索

- 使用 Qdrant Cloud 检索历史案例和知识规则。
- 前台展示 Top 3 引用案例。
- 每条引用支持用户标记：采用、不采用、不相关。
- 引用反馈可下载，后续进入人工引用相关性打分和知识库优化。
- 后台可使用更多知识项辅助生成和审核。

### 7.3 双检双审生成链路

```text
草稿 Agent 生成初稿
↓
一审 Agent 审核草稿
↓
迭代 Agent 根据一审建议修改
↓
终审 Agent 评分
↓
输出最终稿 + 人审意见
```

页面会展示：

- 初稿问题。
- 一审建议。
- 修改后内容。
- 一审到终审的提升维度。

### 7.4 固定审核维度

- 需求匹配
- 品牌安全
- 合规风险
- 事实一致性
- CTA 清晰度
- 内容完整度
- 参考依据

审核维度不让用户选择，避免用户为了高分关闭关键审核项。

### 7.5 PM 后台诊断

普通用户不直接看到低分诊断，只看到发布建议和人审意见。对于低于或等于 7 分的维度，PM 后台会展示：

- 哪个维度低。
- 为什么低。
- 需要用户或团队补充什么资料。
- 内部应该如何沉淀到 Prompt、审核规则或知识库优化。

这让评分从结果展示变成内部质量管理入口，同时避免普通用户看到过多系统实现信息。

## 8. 成功指标

| 指标 | 目标 |
|---|---|
| Top 3 引用返回率 | >= 95% |
| 人工引用反馈覆盖率 | >= 80% |
| Top 3 人工引用相关性平均分 | >= 4/5 |
| 终审平均分 | >= 8 |
| 低分原因解释覆盖率 | 100% |
| 事实一致性通过率 | >= 95% |
| 合规风险通过率 | >= 95% |
| 竞品词命中率 | 0 |
| EDM 合规页脚通过率 | >= 95% |
| 生成成功率 | >= 95% |
| 平均生成耗时 | 持续监控 |

## 9. 非目标

- 不预测真实投放转化率。
- 不替代法务终审。
- 不自动发布内容。
- 不让访问者填写 API key。

## 10. 当前版本结论

当前版本已经实现从输入、RAG、草稿生成、一审、迭代、终审、输出、测评到部署的完整闭环，适合作为 AI 产品经理面试 Demo。
