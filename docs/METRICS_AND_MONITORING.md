# 指标体系与上线监控

## 1. 北极星指标

**可发布内容通过率**

定义：终审评分达到通过阈值，且事实一致性、合规风险、品牌安全、引用依据均通过的内容占比。

这个指标比“生成次数”更接近真实业务价值，因为营销团队最终需要的是可发布内容，而不是任意文本。

## 2. 质量指标

| 指标 | 说明 |
|---|---|
| 终审平均分 | 衡量最终内容整体质量 |
| 一审到终审分数变化 | 衡量迭代 Agent 是否有效 |
| Top 3 引用返回率 | 衡量 RAG 可用性 |
| 引用相关率 | 人工抽检 Top 3 是否与 Brief 相关 |
| 事实一致性通过率 | 是否避免编造参数、认证、保修、用户评价 |
| 合规风险通过率 | 是否避免绝对化、医疗化、EDM 合规缺口 |
| 品牌安全通过率 | 是否避免竞品词和品牌风险 |
| EDM 页脚通过率 | EDM 是否包含 unsubscribe / manage preferences |

## 3. 体验指标

| 指标 | 说明 |
|---|---|
| 生成成功率 | 完整链路成功返回最终稿的比例 |
| 平均生成耗时 | 从点击生成到输出结果的时间 |
| 下载率 | 用户是否下载最终稿 |
| 引用案例查看率 | 用户是否查看 RAG 依据 |
| 审核维度查看率 | 用户是否查看质量评估 |
| 综合评分查看率 | 用户是否关注终审结论 |

## 4. 运营指标

| 指标 | 说明 |
|---|---|
| Blog / EDM 使用比例 | 判断内容类型优先级 |
| 高频产品特性 | 了解用户最常推广的卖点 |
| 高频低分维度 | 指导 Prompt、规则和知识库迭代 |
| API fallback 率 | 判断外部模型服务稳定性 |
| Qdrant fallback 率 | 判断知识库连接稳定性 |

## 5. 上线监控看板建议

第一版看板可以包含：

- 今日生成次数
- 生成成功率
- 平均终审分
- Top 3 引用返回率
- 事实一致性通过率
- 合规风险通过率
- 平均生成耗时
- 低分维度 Top 3

## 6. 迭代判断规则

| 现象 | 可能原因 | 迭代方向 |
|---|---|---|
| 参考依据低分 | 知识库案例不足或检索召回差 | 补案例、优化 payload、调整检索策略 |
| 事实一致性低分 | 模型编造参数 | 加强 Prompt、规则清理、产品事实库 |
| 合规风险低分 | 风险表达没拦住 | 增加规则词表和审核样例 |
| 内容完整度低分 | Blog/EDM 结构不稳 | 补结构模板和格式约束 |
| 生成耗时过长 | 多 Agent 串行调用 | 缓存检索、减少不必要审核、异步化 |
